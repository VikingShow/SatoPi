# 神经突触记忆模型：仿生记忆机制的算法设计

> 本文档回应一个核心问题：能否用算法模拟神经元连接的"用进废退"机制——每使用一次连接加深一次，默认随时间衰减？如何在 AI Agent 的记忆系统中实现这种仿生模型？

---

## 一、神经科学基础：我们要模拟什么？

### 1.1 突触可塑性的两个核心机制

| 机制 | 英文 | 效果 | 触发条件 |
|------|------|------|---------|
| **长期增强（LTP）** | Long-Term Potentiation | 突触连接强度**持久增强** | 突触前后神经元**同时激活** |
| **长期抑制（LTD）** | Long-Term Depression | 突触连接强度**持久减弱** | 长时间**不被激活** |

**Hebb 学习规则（1949）：**

> "Neurons that fire together, wire together."
> 一起激活的神经元会建立连接。

反过来：
> "Use it or lose it."
> 不用的连接会逐渐消退。

### 1.2 生物系统的具体行为

```
时间 t=0:  A ──(w=0.5)──> B     # 初始连接强度 0.5

事件1: A和B同时激活
时间 t=1:  A ──(w=0.65)──> B    # LTP：连接增强

事件2: 再次同时激活
时间 t=2:  A ──(w=0.77)──> B    # 继续增强（但增量递减）

一周没有激活...
时间 t=9:  A ──(w=0.58)──> B    # LTD：自然衰减

一个月没有激活...
时间 t=37: A ──(w=0.21)──> B    # 继续衰减

再次激活
时间 t=38: A ──(w=0.35)──> B    # 重新增强（但不会恢复到最高点）
```

**关键特征：**
1. **增强是对数式的**：越用越强，但边际递减（不会无限增长）
2. **衰减是指数式的**：不用时按指数衰减（开始快后来慢）
3. **有下限阈值**：低于阈值则"突触修剪"（连接消失）
4. **可重新激活**：衰减后的连接再次使用可以恢复（但不完全恢复到历史最高）

## 二、数学模型：如何精确描述这个过程？

### 2.1 基础模型：时间衰减 + 使用增强

```python
# 核心公式
def calculate_weight(w_current, time_since_last_use, was_just_used):
    """
    计算连接权重
    
    参数:
    - w_current: 当前权重 (0.0 ~ 1.0)
    - time_since_last_use: 距上次使用的时间（天）
    - was_just_used: 是否刚刚被使用
    """
    
    # 第一步：自然衰减（指数衰减）
    # λ 是衰减速率，越大衰减越快
    λ = 0.03  # 约23天半衰期
    w_decayed = w_current * math.exp(-λ * time_since_last_use)
    
    # 第二步：如果被使用，增强（对数式增强，边际递减）
    if was_just_used:
        # η 是学习率
        # (1 - w_decayed) 确保越接近1增量越小（饱和效应）
        η = 0.15
        w_new = w_decayed + η * (1 - w_decayed)
    else:
        w_new = w_decayed
    
    # 第三步：阈值检查
    if w_new < PRUNE_THRESHOLD:  # 如 0.05
        return 0.0  # 突触修剪：连接消失
    
    return min(1.0, w_new)  # 不超过上限
```

### 2.2 改进模型：加入使用频率和情境重要性

```python
class SynapticWeight:
    """模拟突触连接权重的完整模型"""
    
    def __init__(self, initial_weight=0.5):
        self.weight = initial_weight
        self.created_at = time.time()
        self.last_accessed = time.time()
        self.access_count = 0
        self.access_history = []  # 记录每次访问时间
        
        # 可调参数
        self.decay_rate = 0.03          # λ: 衰减速率（天^-1）
        self.learning_rate = 0.15       # η: 学习率
        self.prune_threshold = 0.05     # 突触修剪阈值
        self.importance_boost = 1.0     # 重要性加成
    
    def access(self, importance=1.0):
        """
        连接被使用时调用
        
        importance: 使用时的情境重要性
          - 1.0 = 普通使用
          - 1.5 = 在门禁检查中被引用（较重要）
          - 2.0 = 解决了关键 bug（非常重要）
        """
        now = time.time()
        
        # 1. 先执行自然衰减（从上次访问到现在的时间）
        days_since = (now - self.last_accessed) / 86400
        self._apply_decay(days_since)
        
        # 2. 增强：对数式增强 × 重要性系数
        boost = self.learning_rate * importance * (1 - self.weight)
        self.weight = min(1.0, self.weight + boost)
        
        # 3. 更新访问记录
        self.last_accessed = now
        self.access_count += 1
        self.access_history.append({
            'time': now,
            'importance': importance,
            'weight_after': self.weight
        })
    
    def get_current_weight(self):
        """获取当前权重（含时间衰减）"""
        now = time.time()
        days_since = (now - self.last_accessed) / 86400
        
        decayed = self.weight * math.exp(-self.decay_rate * days_since)
        
        if decayed < self.prune_threshold:
            return 0.0  # 已衰减到修剪阈值以下
        return decayed
    
    def _apply_decay(self, days):
        """应用时间衰减"""
        self.weight *= math.exp(-self.decay_rate * days)
    
    def is_alive(self):
        """连接是否仍然存活"""
        return self.get_current_weight() > self.prune_threshold
    
    def half_life_days(self):
        """当前权重的半衰期（天数）"""
        return math.log(2) / self.decay_rate
```

### 2.3 参数调优指南

| 参数 | 含义 | 推荐范围 | 效果 |
|------|------|---------|------|
| `decay_rate` (λ) | 衰减速率 | 0.01-0.1 | 越大遗忘越快 |
| `learning_rate` (η) | 增强速率 | 0.1-0.3 | 越大单次增强越明显 |
| `prune_threshold` | 修剪阈值 | 0.03-0.1 | 越大越容易遗忘 |
| `importance_boost` | 重要性系数 | 1.0-3.0 | 关键知识增强更多 |


## 三、扩散激活（Spreading Activation）：模拟联想记忆

### 3.1 原理

当一个节点被激活时，激活信号沿着连接**扩散**到邻居节点，强度按连接权重和距离衰减。

```
查询："认证"
  → 激活 [认证设计] (activation=1.0)
  → 扩散到 [Redis会话] (activation=1.0 × w=0.8 × decay=0.7 = 0.56)
  → 扩散到 [JWT标准] (activation=1.0 × w=0.9 × decay=0.7 = 0.63)
  → 继续扩散到 [安全策略] (activation=0.63 × w=0.7 × decay=0.7 = 0.31)
  → 激活值低于阈值，停止扩散
```

**和简单关键词搜索的区别：**
- 关键词搜索：只找到包含"认证"的文件
- 扩散激活：找到"认证"以及所有**语义相关**的知识（通过图的连接传播）

### 3.2 算法实现

```python
class SpreadingActivation:
    """扩散激活检索算法"""
    
    def __init__(self, graph, decay_factor=0.7, threshold=0.1, max_depth=3):
        self.graph = graph
        self.decay_factor = decay_factor  # 每跳衰减系数
        self.threshold = threshold         # 激活阈值（低于此停止扩散）
        self.max_depth = max_depth         # 最大扩散深度
    
    def activate(self, seed_nodes, query_context=None):
        """
        从种子节点开始扩散激活
        
        参数:
        - seed_nodes: [(node_id, initial_activation)] 种子节点及初始激活值
        - query_context: 查询上下文（影响激活传播方向）
        
        返回: {node_id: activation_value} 所有被激活的节点
        """
        activations = {}
        
        # 初始化种子节点
        frontier = []
        for node_id, init_activation in seed_nodes:
            activations[node_id] = init_activation
            frontier.append((node_id, init_activation, 0))  # (id, activation, depth)
        
        # BFS 扩散
        while frontier:
            current_id, current_activation, depth = frontier.pop(0)
            
            if depth >= self.max_depth:
                continue
            
            # 获取所有出边
            for edge in self.graph.get_outgoing_edges(current_id):
                neighbor_id = edge.to_node
                
                # 计算传播到邻居的激活值
                # 激活值 = 当前激活 × 连接权重 × 衰减系数
                propagated = current_activation * edge.get_current_weight() * self.decay_factor
                
                # 关系类型加成（因果关系传播更强，提及关系传播较弱）
                relation_boost = self._relation_weight(edge.relation)
                propagated *= relation_boost
                
                # 低于阈值则不继续扩散
                if propagated < self.threshold:
                    continue
                
                # 更新激活值（取最大值，不累加——避免循环放大）
                if neighbor_id not in activations or propagated > activations[neighbor_id]:
                    activations[neighbor_id] = propagated
                    frontier.append((neighbor_id, propagated, depth + 1))
        
        # 按激活值排序返回
        sorted_activations = sorted(activations.items(), key=lambda x: x[1], reverse=True)
        return sorted_activations
    
    def _relation_weight(self, relation):
        """不同关系类型的传播强度"""
        weights = {
            'depends_on': 0.9,    # 依赖关系传播强
            'causes': 0.85,       # 因果关系传播强
            'refines': 0.8,       # 细化关系较强
            'conforms_to': 0.75,  # 约束关系中等
            'relates_to': 0.6,    # 一般关联较弱
            'mentions': 0.4,      # 提及关系最弱
        }
        return weights.get(relation, 0.5)
```

## 四、完整的仿生记忆系统设计

### 4.1 三层协同

```
┌─────────────────────────────────────────────────────────────┐
│ Layer 1: HNSW 分层索引（快速粗定位）                         │
│                                                             │
│  DOMAIN_MAP → 领域INDEX → 文件                              │
│  类比：大脑皮层的功能分区（视觉区/语言区/运动区）           │
│  作用：O(log n) 快速定位到相关区域                          │
└──────────────────────────┬──────────────────────────────────┘
                           ↓ 种子节点
┌──────────────────────────▼──────────────────────────────────┐
│ Layer 2: 扩散激活（精确关联发现）                            │
│                                                             │
│  从种子节点沿图的边扩散                                      │
│  类比：神经元之间的激活传播                                  │
│  作用：发现关键词匹配找不到的语义关联                        │
└──────────────────────────┬──────────────────────────────────┘
                           ↓ 所有激活节点
┌──────────────────────────▼──────────────────────────────────┐
│ Layer 3: 突触权重管理（长期演化）                            │
│                                                             │
│  每次检索使用 → 连接增强（LTP）                             │
│  长期不使用 → 连接衰减（LTD）                               │
│  衰减到阈值以下 → 连接修剪                                   │
│  类比：突触可塑性                                            │
│  作用：系统自动识别"什么知识真正重要"                        │
└─────────────────────────────────────────────────────────────┘
```

```python
def dream_cycle(knowledge_files):
    """定期执行的'做梦'过程——更新所有突触权重"""
    now = time.time()
    
    for file in knowledge_files:
        metadata = read_yaml_header(file)
        synapses = metadata.get('synapses', [])
        
        updated_synapses = []
        for syn in synapses:
            # 计算自然衰减后的权重
            days_since = (now - syn['last_accessed']) / 86400
            decayed_weight = syn['weight'] * math.exp(-DECAY_RATE * days_since)
            
            if decayed_weight < PRUNE_THRESHOLD:
                # 突触修剪：移除这条连接
                log(f"PRUNED: {file} → {syn['target']} (weight={decayed_weight:.3f})")
                continue
            
            syn['weight'] = round(decayed_weight, 3)
            updated_synapses.append(syn)
        
        # 写回文件
        metadata['synapses'] = updated_synapses
        write_yaml_header(file, metadata)
```

**使用时增强：**

```python
def on_knowledge_accessed(file_path, accessed_together_with):
    """当知识被 Agent 引用时，增强相关连接"""
    metadata = read_yaml_header(file_path)
    
    for other_file in accessed_together_with:
        # 查找现有连接
        existing_syn = find_synapse(metadata['synapses'], other_file)
        
        if existing_syn:
            # Hebbian: 一起被引用 → 连接增强
            old_w = existing_syn['weight']
            new_w = old_w + LEARNING_RATE * (1 - old_w)  # 对数增强
            existing_syn['weight'] = round(min(1.0, new_w), 3)
            existing_syn['access_count'] += 1
            existing_syn['last_accessed'] = time.time()
        else:
            # 新连接：首次一起被引用
            metadata['synapses'].append({
                'target': other_file,
                'relation': 'co_accessed',  # 共同引用关系
                'weight': 0.3,              # 初始权重
                'access_count': 1,
                'last_accessed': time.time(),
                'created': time.time()
            })
    
    write_yaml_header(file_path, metadata)
```

### 4.3 Agent 集成方式

```python
class NeuralMemoryRetriever:
    """仿生记忆检索器——给 Agent 用的接口"""
    
    def __init__(self, domain_map_path, knowledge_dir):
        self.domain_map = load_yaml(domain_map_path)
        self.knowledge_dir = knowledge_dir
        self.graph = self._build_graph_from_files()
        self.spreader = SpreadingActivation(self.graph)
    
    def retrieve(self, query, context=None, top_k=5):
        """
        仿生检索：HNSW定位 + 扩散激活 + 突触权重排序
        
        这个方法的输出就是 Agent 需要 Read 的文件路径列表
        """
        
        # Step 1: HNSW 粗定位
        domain = self._match_domain(query)
        seed_files = self._get_domain_entries(domain)
        
        # Step 2: 扩散激活
        seed_nodes = [(f, 1.0) for f in seed_files]
        activated = self.spreader.activate(seed_nodes, query_context=context)
        
        # Step 3: 按 activation × synapse_weight 排序
        results = []
        for node_id, activation in activated[:top_k * 2]:
            node = self.graph.nodes[node_id]
            synapse_weight = node.get_current_weight()  # 突触权重
            
            combined_score = activation * 0.7 + synapse_weight * 0.3
            results.append((node_id, combined_score))
        
        results.sort(key=lambda x: x[1], reverse=True)
        
        # Step 4: 增强被检索到的连接（Hebbian learning）
        retrieved_files = [r[0] for r in results[:top_k]]
        for f in retrieved_files:
            on_knowledge_accessed(f, accessed_together_with=retrieved_files)
        
        return results[:top_k]
    
    def _build_graph_from_files(self):
        """从文件的 YAML 元数据中构建图"""
        graph = MemoryGraph()
        
        for file_path in glob(f"{self.knowledge_dir}/**/*.md"):
            metadata = read_yaml_header(file_path)
            
            node = MemoryNode(
                content=metadata.get('title', ''),
                node_id=file_path,
                metadata=metadata
            )
            graph.add_node(node)
            
            for syn in metadata.get('synapses', []):
                edge = MemoryEdge(
                    from_node=file_path,
                    to_node=syn['target'],
                    relation=syn['relation'],
                    weight=syn['weight']
                )
                try:
                    graph.add_edge(edge)
                except ValueError:
                    pass  # 目标节点尚未加载
        
        return graph
```

## 七、这个模型的本质价值

**不需要人手动维护"什么知识重要"——系统通过使用行为自动学习。**

```
传统方式：
  人定期审查知识库 → 手动标记哪些过时 → 手动删除/归档
  问题：费时、主观、容易遗漏

仿生方式：
  知识被引用 → 自动增强（这个知识确实有用）
  知识不被引用 → 自动衰减（这个知识可能不再需要）
  衰减到阈值 → 自动修剪/归档（系统自动清理）
  问题：可能误删（需要人可以手动干预/恢复）
```

**这就是你说的"神经元的模拟"——不是模拟神经元本身，而是模拟神经元之间的连接强度随使用和时间动态变化的机制。**
